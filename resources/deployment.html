Deployment

To support deployment without requiring you to enter a credit card or running into limits on team sizes, etc. we have our own simple cloud deployment platform csci312.dev. In short, you will push your code to a deployment repository at csci312.dev, which will then build and deploy your application at https://<project name>.csci312.dev, e.g, https://simplepedia.csci312.dev.

  1. As a prerequisite, you should have added an SSH key to GitHub as described in the getting started instructions. Doing so is necessary to obtain deploy permissions. If you attempt the below and get permission-related errors, contact the instructor to make sure your key has been added to the deployment server.
  2. Add the deployment repository as a remote in your repository: ðŸ’» git remote add deploy git@csci312.dev:<project name>, e.g., ðŸ’» git remote add deploy git@csci312.dev:simplepedia. Note that is the name assigned to your group, e.g., "mansfield".
  3. Push the main branch to the deploy repository: ðŸ’» git push deploy main. This should automatically build and deploy your application at https://<project name>.csci312.dev. Any errors should hopefully be reported on in the terminal as the push proceeds.
  4. To add entries to the .env.local file, execute ðŸ’» ssh git@csci312.dev secrets <project name> KEY=value, where again <project name> is the name assigned to your group. KEY=value is the value you want to add or update in the file (without any spaces). For example, ðŸ’» ssh git@csci312.dev secrets simplepedia NEXTAUTH_URL=https://simplepedia.csci312.dev would set the NEXTAUTH_URL key for the simplepedia project.
  5. To view the most recent logs from your application execute ðŸ’» ssh git@csci312.dev logs <project name>. You can optionally add a --lines argument, e.g., --line 50 to look at more lines.
  6. Typically a new deployment will also involve changes to your database schema and or seeding.
